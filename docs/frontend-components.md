# Frontend Components & Hooks

A practical reference for building screens in this project: which components exist, how to
use them, and how the composables ("hooks") work. New to the stack? Start with the
[development-guide.md](./development-guide.md).

Two component families live under `resources/js/Components`:

- **`ui/`** — shadcn-vue primitives (Button, Card, Dialog, Select, …). Generated by the
  CLI, copied into the repo, and yours to edit.
- **Project components** — app-specific pieces built on those primitives (DataTable,
  GroupInput, Multiselect, PageHeader, Modal, the app shell).

> **Import casing:** always `@/Components/...` (capital C). See the development guide.

---

## shadcn-vue primitives (the ones you'll use most)

### Button — `@/Components/ui/button`

```vue
<Button>Save</Button>                          <!-- default -->
<Button variant="outline" size="sm">Edit</Button>
<Button variant="ghost">Cancel</Button>
<Button variant="destructive">Delete</Button>
<Button size="icon-sm" aria-label="Edit"><Pencil class="size-4" /></Button>
<Button as-child><a :href="route('...')">Link styled as button</a></Button>
```

- **variants:** `default`, `destructive`, `outline`, `secondary`, `ghost`, `link`
- **sizes:** `default`, `sm`, `lg`, `icon`, `icon-sm`, `icon-lg`
- `as-child` renders your child element (an `<a>` or Inertia `<Link>`) with button styles.

### Card — `@/Components/ui/card`

```vue
<Card>
    <CardHeader><CardTitle>Basic Information</CardTitle></CardHeader>
    <CardContent>…</CardContent>
</Card>
```

`CardHeader` is a CSS grid — to right-align a header action, override display:
`<CardHeader class="flex items-center justify-end">`.

### Badge — `@/Components/ui/badge`

`<Badge variant="secondary">read</Badge>` — variants: `default`, `secondary`,
`destructive`, `outline`.

### Select — `@/Components/ui/select`

Value is the bound primitive (bind the id/name string, not an object):

```vue
<Select v-model="form.role">
    <SelectTrigger id="role" class="w-full">
        <SelectValue placeholder="Select a role" />
    </SelectTrigger>
    <SelectContent>
        <SelectItem v-for="r in roles" :key="r.id" :value="r.name">{{ r.name }}</SelectItem>
    </SelectContent>
</Select>
```

### RadioGroup — `@/Components/ui/radio-group`

```vue
<RadioGroup v-model="form.gender" class="flex gap-6">
    <div class="flex items-center gap-2">
        <RadioGroupItem id="g-m" value="male" /><Label for="g-m">Male</Label>
    </div>
    <div class="flex items-center gap-2">
        <RadioGroupItem id="g-f" value="female" /><Label for="g-f">Female</Label>
    </div>
</RadioGroup>
```

### Dialog vs AlertDialog

- **Dialog** — general modal (forms). Control with `v-model:open`.
- **AlertDialog** — confirmation prompt with Cancel/Action, used for destructive actions.

Both are used together with the CRUD pattern below.

## Project components

### `<DataTable>` — `@/Components/DataTable/DataTable.vue`

Headless TanStack table rendered with shadcn `Table`. Two modes:

| Mode                | When                          | Extra props                                    |
| ------------------- | ----------------------------- | ---------------------------------------------- |
| Client (default)    | Small lists loaded whole      | —                                              |
| Server (`manual`)   | Large/DB-paginated lists      | `manual`, `:row-count`, `v-model:sorting/pagination`, `:loading` |

**Props:** `columns` (required), `data` (required), `pageSize`, `pageSizeOptions`,
`emptyMessage`, `manual`, `rowCount`, `loading`.

**Columns** are TanStack definitions built with `h()`. Custom cells render badges,
avatars, buttons, etc:

```js
import { h } from 'vue';
const columns = [
    { accessorKey: 'id', header: '#',
      cell: ({ row }) => h('span', { class: 'text-muted-foreground' }, row.getValue('id')) },
    { accessorKey: 'name',                              // sortable header
      header: ({ column }) => h(Button, {
          variant: 'ghost', class: '-ml-3 h-8',
          onClick: () => column.toggleSorting(column.getIsSorted() === 'asc'),
      }, () => ['Name', h(ArrowUpDown, { class: 'size-4' })]) },
    { id: 'actions', enableSorting: false,
      header: () => h('div', { class: 'text-right' }, 'Actions'),
      cell: ({ row }) => h('div', { class: 'flex justify-end gap-1.5' }, [
          h(Button, { variant: 'outline', size: 'icon-sm', onClick: () => openEdit(row.original) },
            () => h(Pencil, { class: 'size-4' })),
      ]) },
];
```

Full server-side contract: [datatable-server-side.md](./datatable-server-side.md).

### `<GroupInput>` — `@/Components/FormInputs/GroupInput.vue`

Label + shadcn Input + error message, with a generated `id` linking label↔input.

```vue
<GroupInput
    legend="Role Name" placeholder="Role" input_type="text"
    v-model="form.name" :required="true" :err_msg="form.errors.name" />
```

**Props:** `legend`, `placeholder` (required); `input_type` (required, any HTML input
type); `input_name`, `required`, `err_msg` (optional). `v-model` binds the value. Passing
`form.errors.<field>` to `err_msg` shows Inertia validation errors inline.

### `<Multiselect>` — `@/Components/FormInputs/Multiselect.vue`

Tag-style multi-select with keyboard support. Works with **option objects**:

```vue
<Multiselect v-model="form.selected_permissions" :options="permissions" />
<!-- options & v-model are arrays of { id, label } -->
```

On submit, map back to what the server expects, e.g.
`data.selected_permissions.map(p => p.label)` (see Roles create/edit).

### `<PageHeader>` — `@/Components/PageHeader.vue`

Page title + an auto-generated breadcrumb derived from the current route.

```vue
<PageHeader title="Manage Roles" user-type="Admin" />
```

The breadcrumb builds itself from `currentRouteName` (`PETAKOM › Section › Page`), with
the intermediate crumbs as clickable Inertia links. `user-type` is accepted for
backward compatibility but no longer part of the trail.

### `<Modal>` — `@/Components/Modal.vue`

Thin wrapper over shadcn Dialog with a title + default slot + close button. Control with
`v-model`:

```vue
<Modal v-model="show" title="Details">Content…</Modal>
```

> For CRUD forms we use the Dialog primitives directly (below) rather than `Modal`, for
> full control over the footer buttons.

### App shell — `Layouts/layouts.vue`

Wraps every page: `Sidebar` (desktop) / `Sheet` (mobile), `Navbar`, content `<slot>`,
`Footer`, and the global `<Toaster>`. Assigned automatically in `app.js` — you don't
import it per page. Nav items live in [`Sidebar.vue`](../resources/js/Components/Sidebar.vue)'s
`navItems` array (route name drives both link and active highlight).

## Toasts

Don't call `toast()` from pages. Redirect from the controller with a flash message and
the layout shows it automatically:

```php
return redirect()->route('...')->with('success', 'Saved!');   // or 'error'
```

For an ad-hoc client-only toast you may `import { toast } from 'vue-sonner'` and call
`toast.success('…')`, but prefer the flash flow for anything tied to a server action.

## Hooks (composables)

Composables are `use*()` functions that package reactive state + behavior for reuse —
Vue's equivalent of React hooks. They live in `resources/js/composables` and are imported
via `@/composables/*`. Full reference: [composables.md](./composables.md).

### `useServerTable` — table state + server reloads

```js
const { sorting, pagination, loading } = useServerTable({
    routeName: 'manage-roles.index',
    paginator: props.roles,     // Laravel paginator payload
    only: ['roles'],            // Inertia partial-reload key
});
```

Bind the three refs to `<DataTable v-model:sorting v-model:pagination :loading>`. The
composable watches sort/page changes and refetches from the server.

### `useDeleteResource` — delete confirmation

```js
const { open: showDelete, target: deleteTarget, deleting,
        ask: askDelete, confirm: confirmDelete } = useDeleteResource('manage-roles.destroy');
```

`askDelete(row.original)` opens the AlertDialog; `confirmDelete()` sends the DELETE and
resets. Aliasing the returns keeps templates readable.

### The standard CRUD page pattern

Every resource index page composes the same pieces:

1. `useServerTable(...)` → drives `<DataTable manual>`.
2. `useForm(...)` inline → a Dialog reused for **both** create and edit (`isEditing`
   toggles `form.post` vs `form.put`, title, and button label).
3. `useDeleteResource(...)` → an AlertDialog confirmation.
4. `columns` built with `h()`, wiring row buttons to `openEdit` / `askDelete`.

Copy [`ManagePermissions/View.vue`](../resources/js/Pages/ManagePermissions/View.vue) as
the simplest working example (single-field form), or
[`ManageRoles/View.vue`](../resources/js/Pages/ManageRoles/View.vue) for one with a
Multiselect and value transform.

### Writing your own composable

- Name it `useX`, put it in `resources/js/composables/useX.js`.
- Use only reactivity primitives (`ref`, `computed`, `watch`); return refs/methods.
- Pass what you need as arguments — don't read a component's `props` from inside.
- Document it in [composables.md](./composables.md).

## Forms with Inertia `useForm`

Pages submit through Inertia, never raw axios:

```js
const form = useForm({ name: '' });

const submit = () => form.post(route('categories.store'), {
    preserveScroll: true,
    onSuccess: () => { form.reset(); showForm.value = false; },
});
```

- `form.processing` → disable the submit button / show "Saving…".
- `form.errors.<field>` → validation errors (feed to `GroupInput :err_msg`).
- `form.transform(fn)` → reshape the payload before send (e.g. permission objects →
  label strings).
- `form.put(route('...update', id))` for edits.
```
